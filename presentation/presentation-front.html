Function Description

handleChange

  • La fonction met à jour les données correspondants aux states firstname, lastname, email, password, confirmPassword.

  • affectation de name et value par décomposition (destructuring) à event.target

  • this.setState permet la mise à jours des state firstname, lastname, email, password, confirmPassword récupéres. La valeur récupérés restera mise à jour au fil de la saisie.

verification

    La fonction contient la logique de vérification des champs avec des regexes avant validation des informations saisies avant le commit au state.

  • une fois le champs valider par la fonction test de regex, on met le state formIsValid à true.

  • S'il ya erreur, elle correspondra au state errors.

handleSubmit

    La fonction permet la soumission du formulaire au server.

  • event.preventDefault() bloque le comportement par défaut l'evènement.

  • Recuperation des states firstname, lastname, email, password par destructuring.

  • Si la fonction verification est true on post les éléments saisies.
    • On utilise axios pour effectuer notre requête, ici envoyer(post) les information récupérer dans la DB.
      • if le post est ok, on met le state redirection à true.
        En cas d'echec on envoie l'erreur.

      • else l'utilisateur n'est pas authenfier on catch l'erreur.

  • On passe formIsValid à true.

render

    Le rendu des éléments avec render

  • if ya un token dans le localStorage on rédirige l'utilisateur au Forum grâce à la méthode Redirect de react-router-dom.

  • if le state redirection est true on rédirige l'utilisateur sur la page connexion.

  • On récupère les keys dans l'objet errors grâce à la méthode Object.keys, et on l'instancie dans le tableau errorArray.

  • On crée un nouveau tableau en utilisant la méthode JavaScript map(), qui contiendra les props les erreurs.

    • Et pour finir on envoie les erreurs detectés dans un paragraphe en cas de besoin.

  • on return les props et states qui seront utilisées par le composant enfant Signup_form.
